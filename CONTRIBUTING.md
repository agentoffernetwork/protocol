# Contributing

Thanks for helping improve AgentOffer Protocol.

## Where To Send Changes

- Editorial fixes, typo fixes, and wording clarifications: direct pull request
- Small non-breaking examples that clarify existing behavior: direct pull request
- New protocol fields, renamed fields, semantic contract changes, or governance changes: open an RFC in `agentoffernetwork/rfcs` first

## Pull Request Expectations

- Keep scope focused
- Explain the user or implementer impact
- Update affected specification documents together
- Add or update examples when behavior changes

## Issue Routing

- `bug`: incorrect or contradictory protocol behavior
- `docs`: wording, structure, navigation, or missing explanation
- `proposal`: candidate changes that may need an RFC
- `question`: clarifications about current semantics

## Code Of Conduct

By participating in this project, you agree to follow `CODE_OF_CONDUCT.md`.
